## Dashboard data routes

The dashboard JS requests these authenticated endpoints with `site`, `period`, and where relevant `granularity` or `limit`:

- `GET /api/analytics/overview`
- `GET /api/analytics/timeseries`
- `GET /api/analytics/breakdowns/<dimension>`
- `GET /api/analytics/bots`

For `/dashboard/all`, every request sends `site=all`. `analytics.reporting.selected_site()` resolves that to no single site, and the reporting querysets aggregate events across the active sites visible to the user. There is no existing endpoint or response shape that returns a list of per-site KPI groups to the page.

## Non-page routes

- `/health/`: database health JSON.
- `/js/script.js`: public browser tracker script.
- `POST /api/events`: browser event collector.
- `POST /api/bot-events`: server-side bot event collector.
